// How it works

Encode. Search. Decode.

RKO separates the problem-independent search from the problem-dependent decoder. Solutions are represented as vectors of continuous random keys in [0,1), enabling a unified optimization engine across any combinatorial problem.

// 01 — ENCODE
Random-Key Representation
Solutions are encoded as vectors χ ∈ [0,1)ⁿ of continuous random keys. This preserves feasibility and simplifies search operators, making them problem-independent.
// 02 — SEARCH
Multi-Metaheuristic Engine
Multiple classic metaheuristics run independently or in parallel, each navigating the random-key space through exploration and exploitation. An elite pool facilitates knowledge sharing.
// 03 — SHARE
Elite Solution Pool
High-quality solutions are stored in a shared elite pool, allowing different metaheuristics to exchange promising solutions and converge toward global optima.
// 04 — DECODE
Problem-Specific Decoder
A decoder translates each random-key vector into a concrete feasible solution for the target problem. Only the decoder needs to be adapted — the rest is reused as-is.

// Quick start

Get started in
minutes.

Implementing RKO for your own problem requires only a custom decoder. The entire optimization engine is reused unchanged.

1. Implement your decoder

Problem.cpp
// Decode a random-key vector // into a feasible solution double decoder( TSol &rk, TProblemData &data ) { // Sort indices by random-key values vector<int> perm = sortByKey(rk); // Build solution from permutation double cost = evaluate(perm, data); return cost; }

2. Run RKO

Terminal
# Clone the repository git clone https://github.com/RKO-solver/RKO_Cpp_v2.0 # Compile make # Run (instance, time) ./runTest instances/name.txt 10

// Proven results

Applied to
NP-Hard problems.

RKO has been benchmarked on challenging NP-hard problems, producing high-quality solutions across diverse combinatorial domains.

🗺️
α-Neighborhood p-Median
Facility location problem seeking to place p facilities minimizing customer assignment costs within neighborhood constraints.
🌳
Tree of Hubs Location
Network design problem connecting demand nodes through a backbone tree of hub facilities, balancing coverage and cost.
🔗
Node-Capacitated Graph Partitioning
Partition a graph into balanced components respecting node capacity constraints while minimizing inter-partition edge costs.
🚚
Middle-Mile Logistics (Amazon)
Real-world application at Amazon: maximizing next-day delivery coverage using RKO with custom local search, achieving +50 bps over baseline.
🤖
Robot Trajectory Planning
Nature-inspired and hybrid quantum approaches for optimizing robot motion trajectories, reducing planning time and energy consumption.
📐
Time-Dependent Traveling Salesman
The TD-TSP has variations in travel times over the planning horizon that are explicitly taken into account in the routing decisions.
🛠️
Job Sequencing and Tool Switching
Determine the best job processing sequence and identify the necessary tools for each job’s processing to minimize tool switches
🪚
Multi-Period Cutting Stock
Determine cutting plans for each period to meet customer demands, where inventory variables link consecutive periods.
🏥
Integrated Operating Room Scheduling
The research addresses surgery scheduling as a combinatorial optimization problem incorporating multi-room scheduling, equipment allocation, and complex availability constraints .
